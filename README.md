## 開発実績 / 担当プロジェクト

### ＜SHOWROOM株式会社＞

#### 課金実装のフルリニューアル対応（約2ヶ月）

**内容**  
2024年のiOS / iPadOS 18リリースを期に、Original StoreKitおよび verify Receipt API がDeprecatedとなったため、サーバーサイドを含めた課金処理のフルリニューアルを実施しました。

事前調査は1名で担当し、クライアント実装方針だけでなく、サーバーサイドを含めたシステム要件の作成まで行いました。  
また、従来未対応であった返金対応の必要性を提案し、不本意な課金損失の防止に貢献しました。

クライアント実装では、課金関連ロジックおよび例外処理を1クラスに集約し、Unit Testの実装まで対応しました。  
すべての例外ケースで原因究明に必要な情報を Firebase Crashlytics に送信する構成とし、問い合わせ・不具合対応の効率向上に寄与しました。

**規模**
- 全2名  
  - iOS：1名  
  - サーバー：1名

#### 配信者ランキング表示のフルリニューアル（約4ヶ月）

**内容**  
既存の配信者ランキング表示が WebView ベースであったため、UI / UX 改善を目的としてネイティブ化を実施しました。

仕様・デザイン作成に約1ヶ月を要し、最終的に18画面の改修および新規実装を行いました。  
実装はすべて SwiftUI + MVVM で行い、Unit Test まで完遂しました。

吹き出しUIやメモリ線付きバーなど、標準コンポーネントで対応できない箇所は SwiftUI の Path を用いて実装しました。

また、プロジェクト規模が大きく Preview のビルド時間が課題であったため、SwiftUI 専用モジュールを作成し、開発効率向上に貢献しました。

**規模**
- 全7名  
  - デザイナー：1名  
  - PM：2名  
  - iOS：1名  
  - Android：1名  
  - Web：1名

### ＜株式会社any style＞

#### 音声通話機能（約2ヶ月）

**内容**  
VTuber とユーザーが1対1で通話する音声通話機能の実装を担当しました。  
通話予約機能、通話待機所機能などを含む合計7画面を SwiftUI + MVVM にて実装しました。

音声通話には Amazon Chime SDK を Swift Package Manager 経由で導入し、不要なビデオ通話・画面共有機能を排除することで、最小限の依存関係で設計しました。

**規模**
- 全3名  
  - iOS：1名  
  - Android：1名  
  - サーバー：1名  

---

#### ボイスコンテンツ販売 / QRコード読み取り機能（約2ヶ月）

**内容**  
VTuber から提供されるボイスコンテンツをアプリ内通貨で購入する機能を実装しました。  
ホーム画面および VTuber プロフィール画面を含む約6画面を SwiftUI + MVVM で実装しました。

QRコード読み取り機能は AVCaptureSession + AVCaptureMetadataOutput を用いて実装し、  
読み取り範囲を rectOfInterest で限定し、枠線は UIBezierPath にて実装しました。

**規模**
- 全3名  
  - iOS：1名  
  - Android：1名  
  - サーバー：1名  

---

#### チャット画面の SwiftUI / MVVM 化（約2ヶ月）

**内容**  
主要機能である VTuber とユーザーのチャット画面を SwiftUI / MVVM にリファクタリングしました。

既存実装では ReSwift および MessageKit を使用していましたが、  
マルチスレッド非対応や実装の複雑化といった課題があり、刷新を行いました。

無限スクロールは ScrollView + LazyVStack を反転させて実装し、  
メッセージ種別は enum に集約することで、可読性および将来的な拡張性を向上させました。

**規模**
- 全1名（iOS）
